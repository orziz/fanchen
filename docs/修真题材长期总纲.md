# 凡尘立道录 · 修真题材长期总纲

> 本文档是本项目在题材定位、世界规则、长期系统演进与高阶内容设计上的主依据。后续 AI 与人工在讨论修真 / 武侠取舍、世界观自洽、系统增层或后期上限时，先以本文为准；若与其他设计文档的局部措辞冲突，以本文的题材口径为准。
>
> 若需要判断“现在该做哪层系统”“某个功能属于哪个阶段”“哪些内容只能先埋种、不能提前完整开放”，继续看 [修真系统分期总表](./修真系统分期总表.md)。

## 1. 最终裁决

- 项目正式选择修真路线，不再把武侠保留为平行主路线。
- 这里的修真不是一开始就脱离地面的高仙侠，而是从凡人社会、地方秩序、产业经营、势力关系和地图推进中长出来的长期修真。
- 项目允许并鼓励未来出现高阶修真内容，但高阶内容必须是对既有系统的升格，不是对既有系统的废弃和替换。
- 本项目的一句话工作口径是：凡人起步，修真升格，地面规则长期有效，后期变强不等于后期脱离世界。

## 2. 为什么选修真，不选武侠

- 选修真，不是因为当前代码和文案已经偏修真，所以懒得改；而是因为本项目追求长期扩展空间、终局上限和系统厚度，这三点更适合修真路线承载。
- 武侠路线更容易在前中期维持地面感和人间秩序，但中后期爽感上限、系统增层空间和题材承载能力更容易见顶。
- 一旦项目后期仍然追求更高维度的成长、宗门、秘境、传承、疆域、地脉与大规模势力博弈，武侠路线常常会再次向超凡漂移，导致题材二次改口。
- 修真路线的风险不在于“会不会失控”，而在于“是否提前定义了高阶修真仍然受哪些世界规则约束”。只要这层规则提前定死，修真路线反而更适合长线维护。

## 3. 最高优先级

后续设计取舍按下面顺序判断：

1. 世界规则自洽。
2. 核心循环连续，不让前期系统在后期失效。
3. 中后期爽感和成长上限。
4. 题材词面的纯度。

解释：本项目可以接受修真不像传统仙侠那样随时飞天遁地，也可以接受高阶修真拥有强烈超凡感；但不能接受世界观与玩法互相拆台，更不能接受前期系统变成后期废件。

## 4. 核心体验定义

### 4.1 玩家起点

- 玩家从寒门凡人或低位起家，不先天站在超凡秩序之外。
- 早期体验围绕求生计、跑门路、攒资产、练根基、找靠山与逐步抬升身位展开。

### 4.2 玩家中期

- 玩家逐步脱离纯生计压力，开始在地方势力、宗门、商路、秘境与产业之间建立自己的盘子。
- 中期的核心不是单纯数值变大，而是从“自己活下去”转向“维持一套能持续运转的结构”。

### 4.3 玩家后期

- 玩家可以进入高阶修真，但后期的核心不是无视一切，而是掌握更大范围、更高层级的秩序。
- 后期要让玩家感到更强、更爽、更能左右局势；同时也要让他承担更大的维护成本、名分压力、资源调度和势力后果。

### 4.4 玩家自由度与世界自演

- 本项目不应把玩家的人生路线锁死成单一主线。跑商、投势、官面差使、宗门修行、游历碰缘、经营置业与人情经营，都应是可成立的中长期路径。
- 不同路径可以有强弱、前置与风险差异，但不应出现“只有主线才算在推进，其他都只是边角料”的结构。
- 世界不应停在原地等玩家触发。NPC、势力、地盘、职位、商路、人情与地方局势，应在日结推进中自行变化，玩家更多是在介入、借势、改向，而不是充当世界唯一发动机。
- 剧情、委托与事件的职责，应是把世界某一时刻的机会、冲突和关系显影出来，而不是变成一套永远摆在面板上的静态待办清单。
- 本项目可以借鉴高自由度历史模拟的“多路线立身”与“世界自推进”体验，但必须继续服从本项目既有的修真经营骨架，不改写“地面规则长期有效”的底层裁决。

## 5. 长期不变的底层规则

以下规则贯穿前中后期，不因境界提升而废除，只会随阶段升格。

### 5.1 地点与路程长期有效

- 地图不是前期教程，也不是后期装饰。地点、路线、区位、邻接关系和行动耗时必须一直有效。
- 前期是一步步走、逐地推进；中后期可以有更快的交通与调度方式；高阶可以有阵门、飞舟、飞遁、跨域通道。
- 但所有高阶移动都必须绑定锚点、成本、冷却、权限或风险，不能把地图抹平成一张纯菜单。

### 5.2 资源与供给长期有效

- 经营、物资、产业、供给链和资源消耗不是低境界阶段的临时玩法，而是整个修真路线的底盘。
- 前期是田产、工坊、铺面、跑商、订单；中后期升格为灵田、丹坊、器坊、矿脉、据点补给、宗门仓储、跨域货路。
- 后期越强，越应该依赖更复杂的供给，而不是越强越不需要供给。

### 5.3 组织与名分长期有效

- 人物不应只靠个人战力推进所有玩法。
- 前期受乡社、商会、官面、行院和地方势力约束；中后期转为宗门、盟约、附庸、分堂、山门、疆域与派系秩序。
- 后期不能变成“个人数值足够大，所以组织失去意义”；正确方向是“个人越强，越需要组织来承接力量”。

### 5.4 身份与代价长期有效

- 变强不等于无代价，修为提升不等于规则免除。
- 前期的代价是金钱、体力、伤势、人情、声望、资格；中后期的代价升级为灵石、材料、时间窗、护持、阵地、名分、盟约、因果和势力风险。
- 高阶玩家做大事时，不能只付出一个抽象数值，要能在世界里找到具体代价来源。

### 5.5 凡俗秩序不会消失，只会退居底层

- 本项目不是纯凡俗武侠，但也不是一旦修真就完全切断凡人世界。
- 凡人社会、地方贸易、治安税赋、世俗人情、基层供给、地面路网始终存在。
- 高阶修真不是站在这些东西之外，而是站在更高层调动、保护、争夺、压榨或维护这些东西。

### 5.6 世界演变长期有效

- 世界状态不能只在玩家点按钮时才变化。势力涨落、人物去留、辖地易手、货路冷热、官面风向与地方关系，应能在玩家旁观时继续推进。
- 玩家的重要性不在于“没有玩家世界就不动”，而在于“玩家能抓住演变中的窗口，把局势往自己要的方向掰过去”。
- 任务、委托和剧情应更像世界演变下的阶段性机会，而不是永驻面板的固定列表；做完的事项应退场，新机会应由世界状态继续生发。
- 任何未来新增系统，只要会把世界重新做成静态菜单、固定刷新表或纯玩家触发器，就默认与本总纲冲突。

## 6. 绝对禁止的题材漂移

- 禁止把后期修真写成纯菜单化成长：闭关一下、领奖一下、全图瞬移一下，前期世界直接失去意义。
- 禁止新增一个完全平行的后期主循环，导致地图、产业、门路、宗门和 NPC 网络都变成废系统。
- 禁止把高阶爽点简单理解为“取消限制”。本项目的高阶爽点应该是“掌握更大秩序”，不是“世界不再约束你”。
- 禁止为了短期爽感，让玩家过早获得不受地点、名分和代价约束的移动、战斗、获取或生产能力。
- 禁止把题材做成纯武侠话语，又在后面硬塞高阶修真，导致终局口径二次漂移。

## 7. 前期系统如何升格，而不是被废掉

| 前期系统 | 后期升格方向 | 禁止的失控写法 |
|---|---|---|
| 地图赶路 | 阵门、飞舟、据点网络、跨域调度 | 后期直接全图点到即达 |
| 地点经营 | 灵田、丹坊、器坊、矿脉、山门供给 | 后期地点只剩进入副本的入口 |
| 跑商与市场 | 跨域货路、战略物资、宗门采购、贡赋交换 | 后期经济只剩卖高阶掉落 |
| 门路与势力 | 盟约、附庸、分堂、区域影响力 | 后期只看个人战力，不再看关系与组织 |
| 宗门成长 | 山门建设、传功体系、外门供给、辖地经营 | 宗门后期只是一个被动加成面板 |
| NPC 关系 | 弟子、执事、客卿、盟友、宿敌、道侣与后代网络 | 后期 NPC 只剩战斗对象或纯文本挂件 |
| 修炼与突破 | 法脉、悟道、闭关、护法、渡险与传承 | 修炼变成脱离地点和资源的纯数值累加 |
| 秘境与战斗 | 护山、争脉、夺宝、域间冲突、镇守与远征 | 秘境后期完全脱离世界，只剩刷本面板 |
| 世俗治理 | 治安、税赋、商路、地权、基层供给 | 后期默认凡俗层面全部无关紧要 |

## 8. 高阶修真的升格原则

### 8.1 增层，不换代

- 后期可以多一层玩法，但不能把前面整层删掉。
- 新层必须吃旧层的资源、地点、组织和关系，旧层也要因为新层变得更重要，而不是更无用。

### 8.2 便利必须绑定锚点

- 所有高阶便利能力都应有世界中的落点，例如山门、阵法、渡口、据点、法坛、飞舟航路、情报节点。
- 没有锚点的便利能力，会直接损坏地图和地点系统。

### 8.3 高阶能力必须有四类限制

1. 成本限制：消耗灵石、材料、药力、人手、护持或时间窗。
2. 风险限制：暴露行踪、引发争夺、惊动敌对势力或伤及自身根基。
3. 名分限制：不是“能做”就“该做”，乱用会坏威望、坏关系、坏秩序。
4. 权限限制：高阶能力应依附于身份、组织、据点、器物或阵地，而非纯个人随身常驻。

### 8.4 高阶爽点应来自影响力，不只来自破坏力

- 后期的爽感不该只靠数值更大、伤害更高、地图更快。
- 更优先的爽点来源是：调动更多人、控制更多地、影响更大范围的供给、守住更重的传承、争夺更高层级的秩序。

### 8.5 高阶内容不能脱离人间

- 就算角色已经站到高阶修真阶段，世界也不能只剩同阶修士和高阶秘境。
- 地方、凡人、基层产业、低阶门路、俗世势力和普通地区仍应持续存在，并成为后期争夺与保护的对象。

## 9. 给后续 AI 的执行规则

### 9.1 做题材判断时，先问这六个问题

1. 这个新内容会不会让地点和路线失去意义？
2. 这个新内容会不会让产业、供给和资源链失去意义？
3. 这个新内容会不会让宗门、势力、门路和名分失去意义？
4. 这个新内容是不是在给后期另起一套平行主循环？
5. 这个新内容的爽点，是掌控秩序，还是取消约束？
6. 这个新内容是否能用现有地图、日结、产业、宗门、NPC 和地点系统承接？

只要其中三项以上的答案偏向“会废旧系统”或“主要靠取消约束来爽”，就应默认重做方案。

### 9.2 默认优先接入的承载骨架

- 地点系统
- 日结与时间推进
- 产业与供给系统
- 势力与宗门系统
- NPC 关系与身份流动
- 现有物品、功法、秘境和世界事件系统

后续 AI 若新增系统，优先想“怎么挂到这些骨架上”，而不是新造平行资源池、平行地图、平行成长线。

### 9.3 默认拒绝的写法

- 纯悬空的高阶修真设定，只负责看起来很强，不负责接现有系统。
- 只靠抽象词汇支撑的修真升级，例如“进入上界后凡间系统全部无效”。
- 让玩家常态化拥有无代价飞遁、无代价跨图、无代价跨层级碾压的能力。
- 为了后期爽点，直接让前期大部分面板沦为摆设。
- 用单独数值或单独资源把高阶阶段封成一款新游戏。

### 9.4 默认鼓励的写法

- 把旧系统升为更大尺度的同类系统。
- 让高阶玩法反过来加强前期资产、据点和组织的价值。
- 用地点、组织、器物、法阵和世界关系来约束高阶便利。
- 让后期冲突围绕灵脉、供给、辖地、传承、分堂和跨域货路，而不只是围绕单人战力。

## 10. 与现有仓库的衔接口径

- [README.md](../README.md) 当前的“修真经营与挂机”表述继续成立，应理解为本总纲下的凡尘修真路线。
- [docs/project-rules.md](./project-rules.md) 中关于地图、产业、宗门、世界系统接入的规则，与本文一致，应继续执行。
- [docs/visual-design-spec.md](./visual-design-spec.md) 与其他纯视觉文档中的题材措辞，默认只解释为视觉气质，不单独构成世界规则依据。
- 若未来其他设计文档出现“修仙”“仙侠”“武侠”等字样，AI 应先判断它说的是视觉风格、宣传口径还是系统规则；只要涉及系统规则，一律回到本文裁决。

## 11. 需要同步更新本文的场景

- 决定新增新的大境界层级，且会改写后期主循环时。
- 决定开放新的高阶移动方式、跨域机制或大型据点体系时。
- 决定让产业、宗门、地图、NPC 或势力中的任一主系统进入新的尺度时。
- 决定修改项目对外题材口径时。
- 决定让玩家后期拥有此前不存在的常态性便利能力时。

## 12. 未来 AI 的最短记忆版

- 本项目选修真，不选武侠。
- 但这里的修真是凡人起步、地面推进、长期升格的修真，不是无视地图和凡俗秩序的高仙侠。
- 高阶内容可以很强，但必须是旧系统的升格版，不能把旧系统废掉重开。
- 地点、资源、组织、名分、代价这五样长期有效。
- 一旦新方案让地图、经营、宗门、门路或 NPC 网络失去意义，就说明题材写歪了。
