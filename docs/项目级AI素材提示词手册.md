## 一、使用原则

### 1. 这份手册解决什么问题

- 保证立绘、背景、图标、地点剪影、整屏 UI 概念图看起来像同一个项目，而不是东拼西凑。
- 保证当前江湖模式先能直接落地，后续再往全项目扩展。
- 把每个资产都整理成固定格式：文件名、用途、尺寸、中文主提示词、负面提示词、重试方向。

### 2. 这套风格的核心判断

- 不是仙侠海报风，不是二次元手游风，也不是写实电影风。
- 是深色水墨武侠：黑、灰、墨白为主，暗金与暗红只点一下，不喧宾夺主。
- 画面要有宣纸、留白、枯笔、飞白、淡墨晕染的东方笔触感。
- 人物、UI、场景必须共享同一套视觉语言：冷、稳、克制、带一点锋利感。

### 3. ChatGPT Images 2.0 通用调用方式

每次生成都按下面四段拼接：

1. 全局总控前缀
2. 具体资产提示词
3. 输出规格
4. 负面提示词

建议直接整段一起喂，不要拆开。

## 二、全局总控前缀

下面这段是所有素材都要带上的总控前缀。

```text
中国古风武侠水墨美术，深色调，黑、灰、墨白为主色，暗金与暗红只作极少量点缀，传统东方美学，江湖气而非仙气，克制、冷峻、留白、锋利。宣纸质感，枯笔飞白，淡墨晕染，笔触细腻但不脏乱，不要现代元素，不要文字，不要水印，不要强动漫感，不要欧美奇幻感，不要过曝高饱和，不要塑料材质感。整体必须像同一个武侠项目的成套素材。
```

### 通用负面提示词

```text
不要现代建筑，不要电线，不要玻璃幕墙，不要机甲，不要枪械，不要赛博元素，不要卡通 Q 版，不要日漫大眼，不要夸张荧光色，不要照片写实，不要西式铠甲，不要宗教圣光，不要花哨 UI，不要海报大字，不要水印，不要签名，不要背景里出现杂乱人物，不要廉价手游感。
```

### 通用输出规则

- 立绘、图标、装饰纹样优先 PNG。
- 需要融入界面的主立绘与局部装饰，优先透明背景。
- 背景大图优先 JPG。
- 如果用于界面背景，默认压低对比度，不抢正文和按钮。
- 如果用于 UI 图标，默认轮廓清晰、内部细节少、缩小后仍能识别。

### 通用重试方向

- 太像仙侠海报：补一句“更像冷峻武侠，不要仙气和梦幻感”。
- 太像二次元立绘：补一句“更接近国画人物立绘，不要动漫五官比例”。
- 太亮太花：补一句“整体更暗、更克制、更少颜色、更少高光”。
- 太脏太糊：补一句“保留水墨笔触，但主体边缘要清晰、层次干净”。
- 不像一套：把“整体必须像同一个武侠项目的成套素材”放到提示词前半段。

## 七、风格一致性终检清单

在把生成好的图片放进项目之前，逐项检查：

- [ ] 所有素材都明显属于同一个深色水墨武侠项目
- [ ] 没有一张图突然偏二次元、偏写实、偏仙侠海报、偏西幻
- [ ] 黑灰墨白仍是主色，暗金和暗红只作点缀
- [ ] 主角立绘边缘清楚，底部透明淡出自然
- [ ] 背景图不会抢正文，不会让白字和金字发虚
- [ ] 图标缩到小尺寸仍能辨认主题
- [ ] 地点剪影不会压过按钮文字
- [ ] NPC 和怪物即使来自不同批次，看起来也像一个世界
- [ ] 整屏 UI 概念图与实际立绘、图标气质一致
- [ ] 没有文字、水印、现代元素、廉价手游感
